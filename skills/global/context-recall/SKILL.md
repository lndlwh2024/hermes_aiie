# Skill: Context Recall

## Trigger

Use this skill when a task may depend on previous decisions, project history, incident lessons, or user preferences.

## Flow

1. Check hard rules and current user instructions first.
2. Read short Memory or project profile if needed.
3. Search project Context Docs only when history is relevant.
4. Return only relevant facts and cite the source document.

## Do Not

- Do not read all historical documents by default.
- Do not let historical notes override current rules.
- Do not store secrets in memory or context docs.
