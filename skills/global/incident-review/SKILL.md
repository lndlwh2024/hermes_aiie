# Skill: Incident Review

## Trigger

Use this skill after a bug, outage, deployment failure, integration failure, or repeated troubleshooting loop.

## Required Output

- Symptom
- Environment
- Root cause
- Fix
- Verification
- Rollback
- Lessons

## Writeback Rule

Short stable facts may go to Memory. Detailed incident reports should go to project Context Docs and the project mirror directory.
