# News Supabase Lessons

## Purpose

Store Supabase database, migration, Edge Function, and environment lessons for the `news` project.

## Current Lessons

- Missing production data can produce behavior differences even when code matches staging.
- `llm_configs` must contain an active LLM configuration for Mode 1 analysis.
- Trigger, `pg_net`, anon key, and internal secret or password-style authorization must be verified per environment.
- Migration repair and timestamp alignment matter when local and remote schema migration histories diverge.

## Loading Rule

Read this file when a task involves Supabase, database migrations, Edge Functions, production/staging drift, or Mode 1/2/3 backend failures.
